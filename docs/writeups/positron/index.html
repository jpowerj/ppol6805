Step 4: Understanding Local vs. Remote

Note that when you first open Positron on your laptop like this, it uses the resources on your local laptop. For example:

  • The displayed files and folders will be files and folders residing on the hard drive of your local machine, and
  • If you run Python or R commands, Positron will execute those using your local R and Python installations

For this reason, we‚Äôll call this Positron ‚Äúrunning locally‚Äù, which is not how we want to use Positron in this course!

Instead, we want to connect Positron to the remote JupyterHub server set up for the course. Once Positron has connected to this server (after you‚Äôve completed the following steps here), we‚Äôll call that Positron ‚Äúrunning remotely‚Äù, since this means that:

  • The files shown in Positron will be files and folders residing on the GIS course server, and
  • If you run Python or R commands, Positron will execute those on the GIS course server, which already has the necessary GIS libraries installed, and also has more computational resources (64 cores, 128 GB of RAM, and 1.5TB of hard drive space) than your local laptop!

When in doubt, one quick way you can tell which ‚Äúmode‚Äù Positron is in (local vs. remote) is to open a new Terminal: In the menubar at the top of Positron, you can click **‚ÄúTerminal‚Äù -> ‚ÄúNew Terminal‚Äù, or you can use the keyboard shortcut Ctrl-Shift-`Ctrl-Shift-`.

If the terminal (which should open in the bottom-left panel of Positron) looks as follows, with a percent sign % right before the portion where you‚Äôre able to start typing, you are likely running locally, since MacOS‚Äôs default Terminal uses this percent sign character % to indicate the start of the command prompt.

If instead the terminal you see looks as follows, with a dollar sign $ right before the portion where you‚Äôre able to start typing, you are likely running remotely, since the Ubuntu operating system installed on the course‚Äôs server uses this dollar sign character $ to indicate the start of the command prompt.

In other words, right now you should see the local version, with the percent sign %. But, after connecting over the following steps, you‚Äôll know you have successfully ‚Äúwarped‚Äù from local to remote Positron when you see the $ in the terminal.

Step 5: Connecting with the ‚ÄúWarp Button‚Äù

In the bottom-left corner of the Positron interface, you should see a‚Ä¶ strange-looking icon (somewhat like a greater-than sign > next to a less-than sign <) that‚Äôs colored differently from the rest of the bottom bar. For example, in this screenshot, the bottom bar is grey but the ‚ÄúWarp Button‚Äù is blue:

Clicking this button is the key step in connecting to the GIS course server! Once you click it, you should see a ‚ÄúCommand Palette‚Äù open near the top of the Positron interface, that looks as follows:

Now, from among the three options shown, please select the Connect Current Window to Host‚Ä¶ option1.

The interface will then prompt you with the text "Enter [user@]hostname[:port]", with a blank text box underneath this prompt. Please enter the following address, where you should replace the YOUR_NETID portion with your actual Georgetown-assigned NetID:

jupyter-YOUR_NETID@guhub.io

So, for example, since my NetID is jj1088, I would enter

jupyter-jj1088@guhub.io

Then press Enter. If the address was entered correctly, you may see the interface look like it‚Äôs ‚Äúrefreshing‚Äù, but then you should be prompted to enter a password:

This is where you should enter the JupyterHub password that was emailed to you near the beginning of the semester. For easy searching in your email client, it was an email with the subject ‚ÄúYour PPOL 6805 / DSAN 6750 JupyterHub Login‚Äù.

Once you have entered (or copied-and-pasted) this password into the text box, press Enter. If the password was entered successfully, you should see a fairly boring interface that looks similar to the local interface, but with the key change that, next to the ‚ÄúWarp Button‚Äù, you should also see the text SSH: guhub.io, as in the following screenshot:

Step 6: Opening Your GUHub Home Folder in Positron‚Äôs Explorer Panel

If you‚Äôve gotten this far, and you see the SSH: guhub.io text in the corner next to the ‚ÄúWarp Button‚Äù icon, you‚Äôre 99% of the way there! All that‚Äôs left is opening your GUHub home folder within Positron‚Äôs Explorer Panel, so that you can access all of the files and folders you‚Äôll be working with during the semester.

To do this, you can either:

  • Click on the ‚ÄúOpen Folder‚Ä¶‚Äù button that appears in the main (upper left) panel of the Positron interface, or
  • Click on the Explorer Panel icon, the icon at the top of the left-hand sidebar in Positron (it kind of looks like the ‚Äúcopy‚Äù icon in a lot of interfaces: two pieces of paper stacked on top of each other), then click the ‚ÄúOpen Folder‚Äù button which appears once the full sidebar panel pops out

In either case, you should now see a Command Palette appear at the top of the interface, this time with:

  • The title ‚ÄúOpen Folder‚Äù, followed by
  • A text box containing a filepath that should look something like /home/jupyter-YOUR_NETID, and finally
  • A dropdown listing of a set of files

Like in the following screenshot:

Don‚Äôt worry about clicking on any of these elements (for example, for sanity purposes, please don‚Äôt click on the HW0 folder just yet). Instead, just press Enter.

After pressing Enter, Positron will likely ask you for your password again. But, once you‚Äôve entered that, the Explorer Panel sidebar should now ‚Äúrefresh‚Äù and should show a listing of the files and folders within your home directory on GUHub, like in the following screenshot:

As you can see in that screenshot, Positron may pop up a box asking you if you trust the authors of the files in this folder. This turns out to be not very much of a security concern, and more of a ‚Äúrelic‚Äù of how Positron would be running if you were running it locally.

Here, since you‚Äôre already connected to the server and running Positron remotely, it‚Äôs basically asking you if you want the files placed on the server to be able to execute code on the server‚Ä¶ which is exactly what you want to do in order to work on the assignments in the course!

I write all that out just to clarify that this popup is not asking whether or not you trust the files to run on your local laptop ‚Äì the execution of all the code, once you‚Äôve connected to the course server using the Warp Button, is carried out on the server itself, so that you don‚Äôt need to worry about e.g. malicious code being run on your laptop!

So, hopefully that eases your mind about this possibly-scary-looking popup, in which you should click the button labeled ‚ÄúYes, I trust the authors‚Äù2.

Once you have approved the message in that popup, you should now (more clearly) see the files in the File Explorer sidebar on the left side of the interface:

Step 8: Disconnecting from the Server

Since you‚Äôve now completed and saved HW0, technically you could just close the Positron window and all would be fine.

At this point, Positron should return you back to its local mode, and now you can close the window without any stress about suddenly disconnecting from the server üòå
